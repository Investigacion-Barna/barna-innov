Cómo evalúa este diagnóstico

1 · Las 8 dimensiones culturales

El instrumento mide ocho dimensiones de la cultura organizacional que influyen en su capacidad de innovar y transformarse digitalmente. Cada dimensión se evalúa con 6 o 7 ítems Likert, para un total de 50 ítems.

Promedio por dimensión

Para cada dimensión D, se calcula la media aritmética de las respuestas Likert de sus ítems:

prom(D) = (1 / n_D) × Σ resp(i) para i ∈ items(D)donde n_D = |items(D)| (6 ítems para la mayoría, 7 para D2 y D6), y resp(i) ∈ {1, 2, 3, 4, 5} es la respuesta Likert del encuestado al ítem i.

El resultado prom(D) ∈ [1, 5] y representa qué tan presente está esa dimensión cultural en la organización.

Los ítems están formulados en sentido positivo: una respuesta de 5 siempre indica mayor cultura innovadora. Esto permite promediar directamente sin invertir escalas.

3 · Del promedio al nivel

Para cada dimensión se promedian sus ítems y el resultado se clasifica en tres niveles según los siguientes umbrales:

Bajo promedio < 2.5
Medio 2.5 ≤ promedio < 3.75
Alto promedio ≥ 3.75

Formalmente, la función de clasificación se define como una función por tramos:

⎧ Bajo si prom(D) < 2.5 nivel(D) = ⎨ Medio si 2.5 ≤ prom(D) < 3.75 ⎩ Alto si prom(D) ≥ 3.75

Los umbrales se calibraron buscando bandas amplias para Medio (donde caen la mayoría de las organizaciones reales) y reservar Alto/Bajo para resultados verdaderamente marcados. Son editables en THRESHOLDS de js/profiles.js si la investigación pide ajustarlos en el futuro.

4 · Los 5 perfiles puros

Cinco perfiles culturales definidos por el marco BARNA. Si las 8 dimensiones de la organización cumplen las condiciones de uno de ellos, se asigna ese perfil.

Cada perfil P se define como un conjunto de condiciones P.targets, donde cada condición es un par (dimensión, nivel esperado). Un perfil matchea cuando todas sus condiciones se cumplen simultáneamente:

matches(P) ⇔ ∀ (d, l) ∈ P.targets : nivel(d) = lEs decir: el nivel real de cada dimensión d coincide exactamente con el nivel esperado l definido por el perfil. Si alguna condición falla, el perfil no matchea y se pasa al siguiente.

Los perfiles se evalúan en orden de especificidad. El primer match gana. Si ningún perfil puro matchea, se pasa al cálculo de mezcla (sección 5).

5 · Cuando no hay un perfil puro

Si ninguna de las 5 reglas matchea exactamente, se calcula la cercanía a cada perfil y se presenta el resultado como una mezcla de los 2 más cercanos.

Paso 1 · Rango y distancia ordinal entre niveles

Asignamos un valor entero a cada nivel para poder medir cuán cerca está un nivel real del esperado:

rango(Bajo) = 0, rango(Medio) = 1, rango(Alto) = 2 dist(a, b) = | rango(a) − rango(b) |Distancia ordinal: 0 si son iguales, 1 si están adyacentes (Bajo↔Medio o Medio↔Alto), 2 si son opuestos (Bajo↔Alto).

Paso 2 · Cumplidos y desviación de un perfil

Para cada perfil P, contamos cuántas condiciones se cumplen y sumamos la distancia de las que no:

cumplidos(P) = #{ (d, l) ∈ P.targets : nivel(d) = l } desviación(P) = Σ dist(nivel(d), l) (d, l) ∈ P.targets con nivel(d) ≠ lcumplidos(P) ∈ {0, 1, ..., |P.targets|}; desviación(P) ∈ {0, 1, ..., 2 × |P.targets|}.

Paso 3 · Score de cercanía

La puntuación final de un perfil pondera los aciertos contra las desviaciones:

score(P) = cumplidos(P) − 0.5 × desviación(P)El coeficiente 0.5 hace que un match exacto valga el doble que una desviación de 1 nivel: prioriza aciertos sin penalizar excesivamente las desviaciones cercanas.

Paso 4 · Selección de la mezcla

Ordenamos los 5 perfiles por score descendente y tomamos los 2 primeros. El primero predomina en la presentación final:

P₁, P₂, P₃, P₄, P₅ = sort(perfiles, key = score, desc) mezcla = (P₁, P₂) // P₁ predomina, con rasgos de P₂

Paso 5 · Porcentaje de coincidencia mostrado

Para cada uno de los 2 perfiles de la mezcla, se muestra al encuestado un porcentaje de coincidencia calculado como:

pct(P) = round( cumplidos(P) / |P.targets| × 100 )Ejemplo: si P tiene 3 targets y cumple 2, pct(P) = round(66.67) = 67%.

Resumen del flujo

El flujo completo para asignar un resultado a un encuestado se puede expresar como:

resultado(encuestado) = perfil_puro si ∃ P : matches(P) mezcla(P₁,P₂) en otro casodonde (P₁, P₂) son los dos perfiles con mayor score(P).

Ejemplo

Una organización con los siguientes niveles:

D1=Medio · D2=Medio · D3=Alto · D4=Bajo · D5=Medio · D6=Medio · D7=Medio · D8=Medio

No matchea ningún perfil puro. Calculando cercanía contra los 5:

  • Cultura reactiva basada en datos (mezcla de Reactiva + Burocracia): score 1.5
  • Burocracia innovadora: matchea D3 Alto pero D4 está Bajo y D5 está Medio (falta D5 Bajo). Score más bajo.

Los 2 perfiles con mejor score se combinan en una tarjeta "A + B" donde el primero predomina. Las descripciones y riesgos de ambos se sintetizan en un solo bloque.

Esto da 25 resultados posibles: 5 perfiles puros + 20 mezclas ordenadas (A predomina, con rasgos de B). En la galería de demos puedes ver un ejemplo de cada uno.
